## đź› ď¸Ź Important Architectural Decisions
*(AI: Never delete these entries. Add to them when making technical choices.)*

**Decision:** Re-introduce ChromaDB Support alongside Neon pgvector via VectorStoreFactory
**Why:** Provide local sandbox flexibility through master control `scale.yaml` and secure configuration. Fallback persists on `chromadata` docker volume ensuring no ephemeral loss. S3 search also added as a placeholder route within the factory pattern.

**Decision:** Rely on Neon pgvector vs ChromaDB  
**Why:** Neon offers persistent, managed infrastructure specifically built for ephemeral cloud functions/containers compared to writing to and saving a local SQLite file (Chroma) reliably across sessions. 

**Decision:** Token-Bucket Throttling with Tenacity  
**Why:** ShopVOX rate limits heavily restrict fast outbound calls (~1 ping per second). A centralized semaphore block wrapper limits requests combined with exponential backoff on HTTP 429s.
