**Pattern: Unsafe State Access**
- **Description:** Agent nodes were attempting to access nested attributes from the state (e.g., `state.address_input`) without first checking if the parent object (`state.lead_context`) existed. This caused `AttributeError` crashes.
- **Resolution:** Implemented defensive coding in all agent nodes. Always check for the existence of parent objects (`if state.lead_context:`) before accessing their children.
- **Applies To:** `liaison_node`, `auditor_node`, `merchant_node`.

**Pattern: Schema Mismatch on Model Instantiation**
- **Description:** The `archivist_node` was directly casting the raw dictionary from a vector search into a Pydantic model (`ProjectRecipe`), assuming the schemas matched. They did not, leading to `ValidationError`.
- **Resolution:** The flow was corrected to use the ID from the search result to call a dedicated factory method (`S3VectorFactory.get_recipe_by_id`) that is guaranteed to return a model-compliant dictionary.
- **Applies To:** Any node that creates a Pydantic model from an external data source.
